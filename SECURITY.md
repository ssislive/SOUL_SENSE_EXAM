# Security Policy

## Reporting Security Issues

The SOUL_SENSE_EXAM project takes security seriously. We appreciate the efforts of security researchers and users who help us keep the project safe.

If you discover a security vulnerability, please **do not report it publicly** by opening an issue.


When submitting a report, please include:
- A detailed description of the vulnerability
- Steps to reproduce the issue
- Affected files, endpoints, or features (if known)
- Screenshots, logs, or proof-of-concept code (if applicable)

## Response Timeline

We aim to handle security reports responsibly and promptly.

- **Acknowledgement:** within **48 hours**
- **Initial assessment:** within **5 business days**
- **Fix & disclosure:** based on severity and impact

We may contact you for additional details during the investigation.

## Supported Versions

Security updates are provided only for the **latest version** of the project available on the main branch.

## Responsible Disclosure

We kindly request that you:
- Do not publicly disclose the vulnerability until it has been addressed
- Allow reasonable time for investigation and remediation

## Recognition

We are happy to acknowledge security researchers for valid reports unless anonymity is requested.

Thank you for helping keep SOUL_SENSE_EXAM secure.
